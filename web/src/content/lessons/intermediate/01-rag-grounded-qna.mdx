# RAG แบบอ้างอิงได้สำหรับ AI Q&A

บทนี้เน้นการออกแบบ Retrieval-Augmented Generation (RAG) สำหรับเว็บความรู้ของทีม โดยมีเป้าหมาย 3 อย่าง:

1. ตอบจากคลังความรู้ที่กำหนดเท่านั้น
2. แสดงแหล่งอ้างอิงทุกครั้งที่ตอบ
3. ลดการตอบที่มั่นใจผิด (hallucination)

## เป้าหมายเชิงคุณภาพ

- **Groundedness**: คำตอบต้องมีหลักฐานจากเอกสารจริง
- **Traceability**: ผู้ใช้ตามกลับไปยัง lesson ต้นทางได้
- **Safety**: ถ้าหาหลักฐานไม่พอ ต้องตอบว่า "ยังไม่พบข้อมูลเพียงพอ"

## สถาปัตยกรรมขั้นต่ำ

1. Ingestion: แปลงบทเรียน MDX เป็น chunk ที่ค้นหาได้
2. Indexing: สร้าง embedding แล้วเก็บใน vector store
3. Retrieval: ดึง chunk ที่เกี่ยวข้องจากคำถาม
4. Generation: ให้โมเดลตอบพร้อมอ้างอิงแหล่งข้อมูล
5. Guardrail: ปฏิเสธคำตอบเมื่อ evidence ต่ำกว่า threshold

## Prompt Pattern ที่แนะนำ

```text
คุณคือผู้ช่วยของแพลตฟอร์มเรียนรู้ AI Agent
ตอบโดยอ้างอิงข้อมูลจาก CONTEXT ที่ได้รับเท่านั้น
ถ้าข้อมูลไม่พอ ให้ตอบว่า "ยังไม่พบข้อมูลเพียงพอในคลังความรู้"
ทุกคำตอบต้องสรุปแหล่งอ้างอิงท้ายข้อความ
```

## Assignment

ออกแบบ API endpoint `/api/assistant` ที่รับคำถาม 1 ข้อ แล้วคืนผลลัพธ์แบบนี้:

- `answer`: ข้อความคำตอบ
- `sources`: รายการเอกสารที่ใช้อ้างอิง
- `confidence`: ค่าความเชื่อมั่น 0-1

## Checklist ก่อนขึ้น Production

- มี citation coverage ขั้นต่ำ 90%
- มี fallback เมื่อ retrieval ว่าง
- มี logging ต่อคำถามและแหล่งข้อมูลที่ใช้
- มี evaluation ชุดคำถามจริงจากผู้ใช้
